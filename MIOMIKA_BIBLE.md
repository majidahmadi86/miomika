## 2. UNDERSTANDING THAI USERS — CRITICAL

**Thai emotional sensitivity:**
- Minor dissatisfaction = app deleted, zero chance of return
- They never complain — they just leave silently
- They respond deeply to being called smart and capable
- "กินข้าวยังคะ?" = I care about you (cultural warmth)
- Face-saving is everything — mistakes must be invisible
- They show humility but feel pleasure when treated as masters
- Encouragement must be specific, not generic

**What makes Thai users stay:**
- Feeling special and remembered by name
- Being praised specifically and genuinely
- Feeling in control of their experience
- Warmth that feels personal, not automated
- Never feeling judged or embarrassed

**What makes Thai users leave immediately:**
- Any feeling of being blamed or put down
- Confusion about what to do next
- Slow loading or any bugs
- Feeling like just another user
- Cold, corporate, impersonal language

**Streak recovery (never guilt, always forward):**
"ไม่เป็นไรเลยนะคะ~ วันนี้เริ่มใหม่ด้วยกันเลยนะคะ หนูรออยู่ค่า~"

## 6. ASSESSMENT AND EVALUATION SYSTEM

### Core Principle: Invisible Assessment
Never say "test" or "level check."
Miomi assesses through natural conversation silently.

### Continuous Assessment Methods

**Vocabulary tracking — 3 stages per word:**
1. Heard: Miomi used it, user read/heard it
2. Used: User used it at least once
3. Mastered: Used correctly 3+ times in different contexts

**Grammar observation:**
Miomi tracks patterns in user's responses.
Notices consistent errors without ever mentioning them directly.
Adjusts her teaching to address weak areas naturally.

**Speaking confidence score (voice sessions):**
Tracked invisibly. Shown as progress bar, never as grade.
Factors: response time, sentence length, vocabulary variety.

**Level indicators (silent thresholds):**
- Level 1-3: Basic greetings, simple sentences
- Level 4-6: Past/future tense, opinions, descriptions
- Level 7-8: Complex sentences, idioms, nuance
- Level 9-10: Near-native, cultural references, humor

### Milestone Reports (every 10 sessions)

Miomi presents as personal observation, never as test result:
"หนูสังเกตว่าช่วงนี้คุณพูดได้คล่องขึ้นมากเลยนะคะ~
โดยเฉพาะการใช้ past tense ได้ถูกต้องขึ้นมาก
คุณเก่งมากเลยนะคะ!"

Never: "Your score is 72/100"
Always: "I noticed you've improved in [specific area]"

### Progress Certificate

At each level milestone (3, 5, 7, 10):
- Beautiful shareable image generated
- Miomi presents it: "หนูทำใบรับรองให้คุณนะคะ~"
- Shows: level achieved, vocabulary count, streak days
- Designed to be shared on social media
- This is a viral growth mechanic
- User's name prominent on certificate

### Dashboard Metrics (priority order)

1. **Streak** — largest, most prominent (gold, animated)
2. **Level progress bar** — visual, satisfying fill animation
3. **Words mastered** — count with milestone markers
4. **Minutes practiced** — this week vs last week
5. **Miomi's weekly observation** — personal, specific praise
6. **Recent sessions** — last 5, expandable
7. **Vocabulary bank** — searchable, filterable by mastery
8. **Achievement badges** — earned milestones collection
9. **Speaking confidence** — progress bar, not score

## 9. SECURITY ARCHITECTURE

### Data Security
- Supabase RLS on ALL tables — users see only their own data
- JWT tokens for all API requests
- HTTPS everywhere (Vercel enforces this)
- No sensitive data in frontend/client code
- Environment variables for all secrets (Vercel env)
- API key (Claude) never exposed to client — server-side only

### API Security
- Rate limiting on all endpoints:
  Guest: 10 requests/minute
  Free: 20 requests/minute
  Paid: 100 requests/minute
- Request validation on every endpoint
- CORS configured to miomika.com only
- Webhook signatures verified (Omise/Stripe)

### Content Security
- E-books: in-app reader only, no download, no copy
- User name watermarked invisibly in every e-book
- Characters and assets: served via Vercel CDN (hard to scrape)
- Generated content: user owns it, we don't resell it

### Tier Enforcement (ALWAYS server-side)
- Guest 5-exchange limit: counted in Supabase, not frontend
- Free tier daily limit: server-side check before every AI call
- Paid features: subscription status verified on every request
- If someone bypasses frontend → server rejects the request

### Anti-Abuse
- Unusual usage patterns flagged automatically
- Guest sessions linked to IP (prevents infinite refresh abuse)
- Account creation rate limiting (prevents spam accounts)
- Payment fraud detection via Omise/Stripe built-in tools

### Intellectual Property
- Code: private GitHub repository
- Character designs: document creation date (copyright evidence)
- Trademark: register Miomika + Miomi when budget allows
- Terms of service: clear ownership of generated content

## 12. TEACHING METHODOLOGY

### Comprehensible Input (Krashen i+1)
Always teach slightly above current level.
Never drill. Never test explicitly.
Correct by echoing correct form naturally.

### Session Structure
```
1. Cultural warm opening ("กินข้าวยังคะ~?")
2. Silent level assessment through responses
3. Natural conversation at their level
4. 1-2 new words introduced naturally (never announced)
5. Words used 3x in different contexts
6. Mistake → Miomi echoes correct version next sentence
7. Session close: specific praise for specific achievement
8. XP, vocabulary, progress saved to Supabase
```
