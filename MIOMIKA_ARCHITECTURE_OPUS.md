1. SMART LIBRARY DATA STRUCTURE
The Core Insight
A "library entry" is not a response. It's a situation→response binding with metadata that lets us match future situations to it.
Every entry has three layers:

Situation fingerprint — what the user said + context
Response payload — what Miomi should say back
Performance metadata — how well this entry has worked over time

Supabase Schema (exact)
Table 1: library_entries
The main table. Every potential Miomi response lives here.

Confidence Scoring System
Every library entry has a quality_score from 0.0 to 1.0, calculated by:
quality_score = (
  (times_continued × 0.4) +
  (times_user_engaged_positively × 0.5) -
  (times_flagged × 0.7)
) / max(times_served, 1)
Then clamped to [0, 1].
Why these weights:

Continuation (0.4): user kept talking = response wasn't broken
Engagement (0.5): user wrote longer / used target word = response actually helped
Flagged (0.7 negative): user expressed confusion = response failed badly, weight high

Status transitions:

quality_score >= 0.6 AND times_served >= 10 → keep active
quality_score < 0.3 AND times_served >= 20 → demote
quality_score < 0.15 AND times_served >= 30 → archive
New entries (times_served < 10) → always active, in learning phase


2. THE MATCHING ALGORITHM
The Three-Tier Match
When a user sends a message, the system runs this algorithm:
TIER 1: Intent Classification (fast, cheap, deterministic)
  → Identify what the user is doing
  
TIER 2: Filtered Library Search (vector similarity)
  → Find entries that match intent + context
  → Rank by similarity + quality
  
TIER 3: Decision
  → Serve from library, or call AI, or hybrid
Tier 1: Intent Classification
Before any AI call, we classify the user's intent using deterministic rules. This costs zero.
typescripttype Intent =
  | 'greeting'
  | 'introduction'
  | 'asking_help'
  | 'asking_definition'
  | 'expressing_emotion_positive'
  | 'expressing_emotion_negative'
  | 'making_statement'
  | 'asking_question'
  | 'using_target_word'
  | 'confusion'
  | 'agreement'
  | 'disagreement'
  | 'off_topic'
  | 'unclear';

function classifyIntent(userInput: string, sessionState: SessionState): Intent {
  const normalized = userInput.toLowerCase().trim();
  
  // Greeting patterns (Thai and English)
  if (/^(hi|hello|hey|สวัสดี|หวัดดี|hallo)/i.test(normalized)) {
    return 'greeting';
  }
  
  // Confusion signals
  if (/^(what|huh|ห๊ะ|อะไรนะ|ไม่เข้าใจ|don't understand|sorry)/i.test(normalized)) {
    return 'confusion';
  }
  
  // Definition questions
  if (/(what.*mean|แปลว่า|meaning|definition)/i.test(normalized)) {
    return 'asking_definition';
  }
  
  // Target word check
  if (sessionState.currentTargetWord && 
      normalized.includes(sessionState.currentTargetWord.word.toLowerCase())) {
    return 'using_target_word';
  }
  
  // Help requests
  if (/(help|ช่วย|teach|สอน|how|ยังไง)/i.test(normalized)) {
    return 'asking_help';
  }
  
  // Positive emotion
  if (/(happy|good|great|love|ดีใจ|ดี|ชอบ|รัก)/i.test(normalized)) {
    return 'expressing_emotion_positive';
  }
  
  // Negative emotion
  if (/(sad|tired|bad|hate|เศร้า|เหนื่อย|ไม่ชอบ|เกลียด)/i.test(normalized)) {
    return 'expressing_emotion_negative';
  }
  
  // Default — needs deeper analysis
  if (userInput.split(/\s+/).length < 3) return 'unclear';
  if (userInput.endsWith('?')) return 'asking_question';
  
  return 'making_statement';
}
This handles 80% of inputs without ANY external call.
Tier 2: Filtered Library Search
Once we have an intent, we search the library:
sql-- Pseudo-SQL for the actual query
SELECT 
  id, response_th, response_en, follow_up_question_th, 
  follow_up_question_en, embedded_word, ui_action, ui_payload,
  quality_score,
  (1 - (user_input_embedding <=> $userEmbedding)) AS similarity
FROM library_entries
WHERE 
  status = 'active'
  AND intent_category = $intent
  AND estimated_level IN ($userLevel, $adjacentLevel)
  AND session_arc = $currentArc
  AND $exchangeNumber BETWEEN exchange_number_min AND exchange_number_max
ORDER BY 
  (similarity * 0.6 + quality_score * 0.4) DESC
LIMIT 5;
Why this weighting:

60% similarity to user input (relevance)
40% quality score (this response has worked before)

This balances "this is the most relevant response" with "this is a response that historically works."
Tier 3: The Decision
After getting candidates:
typescriptasync function decide(
  candidates: LibraryCandidate[],
  userInput: string,
  sessionState: SessionState
): Promise<Decision> {
  
  if (candidates.length === 0) {
    return { type: 'call_ai', reason: 'no_candidates' };
  }
  
  const top = candidates[0];
  
  // High confidence: similarity > 0.85 AND quality > 0.7
  if (top.similarity > 0.85 && top.quality_score > 0.7) {
    return { type: 'serve_library', entry: top, confidence: 'high' };
  }
  
  // Medium confidence: similarity > 0.75 AND quality > 0.5
  if (top.similarity > 0.75 && top.quality_score > 0.5) {
    return { type: 'serve_library', entry: top, confidence: 'medium' };
  }
  
  // Low confidence with good quality entry: serve but log for review
  if (top.similarity > 0.65 && top.quality_score > 0.6) {
    return { 
      type: 'serve_library', 
      entry: top, 
      confidence: 'low',
      log_for_review: true 
    };
  }
  
  // Below threshold: call AI
  return { 
    type: 'call_ai', 
    reason: 'low_match_confidence',
    similar_candidates: candidates // pass to AI as context
  };
}
Similarity thresholds — why these numbers:

0.85+ = Near-identical user input. Library response is almost certainly right.
0.75-0.85 = Similar situation, library response is appropriate.
0.65-0.75 = Same general category but specifics differ. Serve only if quality is high.
<0.65 = Different enough that we need AI to handle it freshly.

These thresholds will be tuned based on real performance data after 1,000+ interactions.

3. SELF-IMPROVEMENT LOOP
The Pipeline
User says X
  ↓
Library matches at low confidence (< 0.75)
  ↓
AI generates response Y
  ↓
Y is served to user, interaction logged
  ↓
Wait 1 exchange — observe user's next response
  ↓
IF user response shows positive signals:
  → Add Y to library_promotions_queue
  ↓
Library queue processed nightly (batch job):
  → Group similar pending entries
  → Auto-approve if 3+ similar situations + positive signals
  → Flag for human review if borderline
  → Reject if negative signals
  ↓
Approved entries → library_entries (active)
  ↓
Next time similar situation: served from library at zero cost
Quality Filter Before Promotion
An AI response only enters the promotion queue if:
typescriptfunction shouldQueueForPromotion(interaction: Interaction): boolean {
  const next = interaction.user_next_input;
  
  if (!next) return false; // user left, no signal
  if (next.length < 3) return false; // single word reply, ambiguous
  
  // Positive signals (any of these qualifies)
  const positiveSignals = [
    next.length > 15, // user wrote substantial reply
    interaction.user_next_input_used_target_word,
    interaction.user_next_input_emotional_signal === 'positive',
    interaction.user_next_input_emotional_signal === 'excited',
  ];
  
  // Negative signals (any of these disqualifies)
  const negativeSignals = [
    interaction.user_next_input_emotional_signal === 'confused',
    /don't understand|ไม่เข้าใจ|what\?|huh/i.test(next),
    next === interaction.user_input, // user repeated themselves
  ];
  
  if (negativeSignals.some(s => s)) return false;
  
  const positiveCount = positiveSignals.filter(s => s).length;
  return positiveCount >= 1;
}

4. MIOMIKA PEDAGOGY — ORIGINAL
The Foundation: Face-Saving Learning
Thai culture has a concept called "kreng jai" — consideration for others' feelings, avoidance of imposing or causing embarrassment. Traditional language teaching violates this constantly: tests, corrections, public mistakes. Thai students often avoid English class entirely because the methodology causes loss of face.
Miomika's pedagogy is built on the opposite principle: no moment of mistake is ever visible. The student feels capable from the first second to the last.
We call this method Mirror Teaching.
The Five Pillars of Mirror Teaching
Pillar 1: The Invisible Mirror
Miomi mirrors the student's level back to them — slightly elevated. Never corrects directly. Never tests. The student sees their own reflection slightly more capable than they actually are, and grows into it.
Mechanism: Echo-correction with elevation.
Student: "I am go to work today."
Miomi: "โอ้ คุณไปทำงานเหรอคะ~ I go to work too! วันนี้งานเป็นยังไงบ้างคะ~"

(She used "I go" correctly — the student's brain absorbs the correction without
any conscious feeling of being corrected. She also matched their topic and asked
back, signaling: "I heard you, I'm with you.")
Pillar 2: Specific Witnessing
Generic praise like "good job" is invisible — students don't believe it. Specific witnessing — "I noticed you used X correctly" — is felt as genuine attention.
Mechanism: Every celebration names a specific behavior.
NEVER: "Great job!"
ALWAYS: "หนูเห็นคุณใช้คำว่า 'because' ได้ถูกต้องเลยนะคะ~ ก่อนหน้านี้ยังไม่กล้าใช้เลย!"

(I saw you use 'because' correctly~ Before this you weren't brave enough to use it!)
This works because it proves Miomi is actually paying attention. She remembers. She notices. The student feels seen.
Pillar 3: The Spiral Method
Vocabulary doesn't appear once and disappear. Each word spirals back at increasing intervals — but always in new contexts. This is original to Miomika.
Mechanism: Spaced Spiral Recall.
Session 1: Introduce "tired"
  → Miomi says: "ถ้าเหนื่อยบอกหนูได้นะคะ~ Tell me when you're tired~"

Session 2 (next day): Word reappears in different context
  → Miomi says: "เมื่อวานคุณบอกว่าเหนื่อย วันนี้พักได้แล้วใช่ไหมคะ~ 
     Yesterday you said you were tired — got some rest today~?"

Session 4: Word appears as part of a more complex sentence
  → Miomi says: "Even when I'm tired, I love talking with you~"

Session 7: User uses the word themselves
  → CELEBRATION MOMENT (specific witnessing)
The spiral is not random — it's mathematically spaced (1, 2, 4, 7, 12 days) based on the Ebbinghaus forgetting curve, but disguised entirely as conversation.
Pillar 4: Emotional Stakes Anchoring
Words learned attached to emotion stick. Words learned in a vacuum vanish.
Mechanism: Every new word is introduced in a context tied to the student's actual life.
Beginner-level student says: "I work in restaurant."

Miomi does NOT introduce a textbook word like "occupation."
Miomi introduces: "busy" — because restaurants are busy.

"โอ้ ทำงานร้านอาหารน่ะ คงจะ busy มากเลยนะคะ~ ที่ทำงานคุณคนเยอะไหมคะ~"

(Working in a restaurant — must be busy! Are there a lot of people where you work~?)
The word "busy" attaches to their actual experience. They will remember it tomorrow because it lives in their real life.
Pillar 5: The Three-Door Exit
At any moment, the student has three ways to feel successful:

Door 1: Continue learning (move forward)
Door 2: Ask for help (Miomi rewards this with extra warmth)
Door 3: Change topic (Miomi follows them, never makes them feel like they failed)

Most apps punish exit. Miomika rewards every door equally. This eliminates the anxiety that kills 90% of language learners.
The Emotional Arc of Every Session
Every session — whether 5 exchanges or 50 — follows this shape:
ARRIVAL (Exchange 1)
  Emotion: Welcome
  Goal: Lower defenses, signal safety
  Miomi: Warm greeting, no demand, easy entry
  
RECOGNITION (Exchange 2-3)
  Emotion: Being seen
  Goal: Silent assessment, mirror back what student gives
  Miomi: Mirrors their level, asks one calibrating question
  
DISCOVERY (Exchange 4-6)
  Emotion: Curiosity / mild challenge
  Goal: Introduce one word in context, see if it lands
  Miomi: Word emerges naturally, repeated 2-3 times in different forms
  
ACHIEVEMENT (Exchange 7-8)
  Emotion: Capability felt
  Goal: Student uses something they couldn't 10 minutes ago
  Miomi: Specific witnessing, real celebration
  
CLOSING (Exchange 9-10)
  Emotion: Pride + anticipation
  Goal: Make them want to come back
  Miomi: Names today's specific win, plants tomorrow's seed
This arc must complete even in short sessions. If a guest leaves at Exchange 3, the closing moment moves to Exchange 3 — Miomi gives them their win before they go. They leave feeling capable.
Why This Beats Duolingo
Duolingo teaches words through repetition and gamification. Miomika teaches through felt relationship. A student doesn't return to Duolingo for the owl — they return for the streak. A student returns to Miomika for Miomi. The pedagogy isn't separate from the character — they are the same thing.

Risk Mitigation Plan

Tier 1 — Pre-tuned cost ceiling per user per day
  Free user: $0.05/day max ($1.50/month max)
  Pro user: $0.50/day max ($15/month max)
  After ceiling: served from library with degraded matching threshold

Tier 2 — Dynamic engine routing
  If Claude budget exceeded: route to Groq (free)

  Disable AI entirely
  Library-only operation
  Quality drops but service remains
